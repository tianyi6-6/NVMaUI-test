# 电源控制面板重构说明

## 概述

本次重构将原来的模拟电源控制面板升级为支持实际设备控制的版本，支持以下设备：

- **Rigol DP832**: 三通道可编程直流电源
- **UNI-T UDP3305S**: 三通道可编程直流电源

## 主要改进

### 1. 实际设备控制
- 移除了随机数据填充
- 实现了与真实设备的通信
- 支持电压、电流的实时设置和读取
- 支持输出开关控制

### 2. 设备连接管理
- 异步设备连接，避免UI阻塞
- 连接状态实时显示
- 自动错误处理和重连机制
- 设备断开时的安全处理

### 3. 用户界面改进
- 添加了连接状态指示器（绿色/红色圆点）
- 设备连接状态标签
- 控件状态与连接状态联动
- 错误信息实时显示

### 4. 错误处理
- 完善的异常捕获和处理
- 详细的错误日志记录
- 用户友好的错误提示
- 设备操作失败时的状态恢复

## 配置文件说明

### power_config.ini 结构

```ini
[Setup]
power_number = 2                                    # 电源设备数量
power_device_type_1 = UNI-T-UT33C                 # 设备1类型
power_device_type_2 = Rigol-DP832                 # 设备2类型
power_device_id_1 = USB0::0x0483::0x5740::ADP3524510002::INSTR  # 设备1资源名
power_device_id_2 = USB0::0x1AB1::0x0E11::DP8C263202813::INSTR  # 设备2资源名

[power_1]
name = NV磁传感器供电电源                          # 设备名称
channels = 3                                       # 通道数量
ch1_name = NV磁传感器 - 样机1                      # 通道1名称
ch1_max_volt = 30                                 # 通道1最大电压
ch1_max_current = 3                               # 通道1最大电流
ch1_default_volt = 24                             # 通道1默认电压
ch1_default_current = 3                           # 通道1默认电流
ch1_editable = False                              # 通道1是否可编辑
# ... 其他通道配置
```

## 使用方法

### 1. 环境准备
确保已安装以下依赖：
```bash
pip install PySide6 pyvisa
```

### 2. 设备连接
1. 将电源设备通过USB连接到计算机
2. 确保设备驱动已正确安装
3. 运行测试脚本检查设备识别：
   ```bash
   python test_power_panel.py
   ```

### 3. 配置文件设置
1. 编辑 `config/power_config.ini`
2. 设置正确的设备类型和资源名
3. 配置各通道的参数

### 4. 启动程序
```bash
python power_panel.py
```

### 5. 操作步骤
1. 勾选"Enable Communication"启用设备通信
2. 等待连接状态变为"Connected"（绿色）
3. **设备连接成功后会自动按照配置文件中的默认参数设置电压和电流**
4. 设置电压和电流值（如果通道可编辑）
5. 点击"ON/OFF"按钮控制输出
6. 实时监控电压、电流和功率读数

## 功能特性

### 实时监控
- 每秒更新一次设备读数
- 实时显示电压、电流、功率
- 自动计算功率值

### 自动默认参数设置
- 设备连接成功后自动设置默认电压和电流
- 从配置文件中读取默认参数
- 避免重复设置，提高操作效率
- 确保设备启动时处于安全状态

### 安全保护
- 电压和电流限制检查
- 设备连接状态验证
- 操作失败时的状态恢复

### 日志记录
- 所有操作自动记录到日志文件
- 时间戳和详细信息
- 错误信息完整记录

### 多设备支持
- 支持同时控制多个电源设备
- 每个设备独立配置和控制
- 设备间互不干扰

## 故障排除

### 常见问题

1. **设备连接失败**
   - 检查USB连接
   - 确认设备驱动安装
   - 验证资源名是否正确

2. **VISA资源未找到**
   - 安装NI-VISA或PyVISA
   - 检查设备管理器中的设备状态

3. **权限错误**
   - 以管理员身份运行程序
   - 检查设备访问权限

4. **通信超时**
   - 检查设备是否被其他程序占用
   - 重新连接设备

### 调试模式
在代码中设置 `debug=True` 可以启用详细的通信日志：
```python
controller = RigolDP832Controller(resource_name, debug=True)
```

## 技术架构

### 类结构
- `PowerPanel`: 主界面类
- `PowerDevice`: 设备管理类
- `PowerChannel`: 通道控制类
- `DeviceConnectionThread`: 设备连接线程

### 线程安全
- 设备连接使用独立线程
- UI更新通过信号槽机制
- 避免界面阻塞

### 错误处理
- 分层错误处理机制
- 用户友好的错误提示
- 自动状态恢复

## 扩展性
