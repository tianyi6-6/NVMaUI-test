# 航迹采集系统 - 导入和编辑功能说明

## 新增功能

### 3. 初始化默认航线功能

**功能描述：** 以实验站为中心，自动生成3x3网格的9个默认航点，方便用户快速开始航迹规划。

**使用方法：**
1. 先设置实验站位置
2. 点击"初始化默认航线"按钮（紫色按钮）
3. 系统会自动生成9个航点，形成3x3网格
4. 每个航点间距约100米
5. 时间戳按5分钟间隔递增

**特点：**
- 以实验站为中心生成网格
- 自动计算经纬度偏移（考虑地球曲率）
- 每个航点都有默认描述
- 激光测距仪默认为未启动状态（-1）
- 时间戳从当前时间开始，每5分钟递增

**注意事项：**
- 必须先设置实验站位置
- 会清空现有的所有航点
- 生成的航点可以后续编辑修改

### 1. 导入航迹功能

**功能描述：** 从JSON文件导入航点数据，支持批量导入和实验站设置。

**使用方法：**
1. 点击"导入航迹"按钮（蓝色按钮）
2. 选择要导入的JSON文件
3. 系统会自动解析文件并导入航点
4. 如果文件中包含实验站信息，会询问是否设置实验站

**JSON文件格式：**
```json
{
  "NV Mag Station": {
    "名称": "实验站名称",
    "纬度": 29.559678,
    "经度": 119.176676
  },
  "航点数据": [
    {
      "名称": "Point 1",
      "时间戳": "2024-01-15 10:30:00",
      "纬度": 29.560000,
      "经度": 119.177000,
      "激光测距仪(m)": 150.5,
      "描述": "起始点"
    }
  ]
}
```

**注意事项：**
- 文件必须包含"航点数据"字段
- 航点数据必须是数组格式
- 时间戳格式：YYYY-MM-DD HH:MM:SS
- 激光测距仪距离为-1表示未启动

### 2. 编辑航点功能

**功能描述：** 编辑现有航点的所有信息，使用与新增航点相同的界面。

**使用方法：**
1. 在航点表格中选择要编辑的航点
2. 点击"编辑航点"按钮（橙色按钮）
3. 在弹出的对话框中修改航点信息
4. 点击"确定"保存修改

**编辑内容：**
- 时间戳（支持日历选择）
- 纬度（度分格式）
- 经度（度分格式）
- 激光测距仪距离
- 描述信息

**特点：**
- 对话框会显示当前航点的所有值作为默认值
- 支持度分格式的经纬度输入
- 时间戳支持日历选择器
- 激光测距仪距离范围：-1到10000米

### 4. 经纬度编辑限制

**功能描述：** 禁用表格中经纬度列的直接编辑功能，确保坐标修改的准确性。

**限制内容：**
- 纬度列（第3列）不可直接编辑
- 经度列（第4列）不可直接编辑
- 双击这些列会显示提示信息

**编辑方式：**
- 使用"编辑航点"按钮进行经纬度修改
- 支持度分格式的精确输入
- 避免直接编辑可能导致的格式错误

**其他列编辑：**
- 时间戳列：支持双击编辑和格式验证
- 激光测距仪列：支持直接编辑和范围验证
- 描述列：支持直接编辑

## 错误处理

### 导入错误：
- 文件不存在：显示错误信息
- JSON格式错误：显示具体错误位置
- 数据格式错误：跳过错误数据，继续导入其他数据
- 缺少必要字段：显示警告信息

### 编辑错误：
- 未选择航点：显示警告信息
- 输入验证：自动验证数值范围和时间格式
- 数据恢复：输入错误时自动恢复原值

## 兼容性

- 支持之前保存的JSON文件格式
- 向后兼容，不影响现有功能
- 新增字段有默认值处理

## 注意事项

1. 导入功能会清空当前所有航点
2. 编辑功能需要先选择航点
3. 时间戳格式必须严格遵循YYYY-MM-DD HH:MM:SS
4. 激光测距仪距离不能小于-1
5. 导入时会自动创建保存目录（如果不存在）
6. 初始化默认航线功能已修复递归调用问题

## 崩溃修复 - 安全版本

### 问题描述
初始化默认航线功能和其他操作在某些情况下会导致程序崩溃，原因是UI回调的递归调用和同步更新导致的死锁。

### 安全版本修复方案
1. **使用QTimer延迟执行**：避免UI阻塞和递归调用
2. **分离表格更新和回调触发**：使用`_safe_update_table`方法
3. **延迟触发地图更新**：使用QTimer.singleShot避免同步调用
4. **异常处理机制**：确保程序稳定性
5. **信号重连保护**：防止信号连接丢失

### 安全版本实现
```python
def init_default_waypoints(self):
    """初始化默认航线（3x3网格）- 安全版本"""
    # 使用QTimer延迟执行，避免UI阻塞和递归
    from PySide6.QtCore import QTimer
    
    def create_default_waypoints():
        try:
            # 执行初始化操作
            # ...
            # 安全地更新表格（不触发回调）
            self._safe_update_table()
        except Exception as e:
            QMessageBox.critical(self, '初始化失败', f'创建默认航点时发生错误:\n{str(e)}')
    
    # 使用QTimer延迟执行，避免UI阻塞
    QTimer.singleShot(100, create_default_waypoints)

def _safe_update_table(self):
    """安全地更新表格，不触发回调"""
    # 手动触发一次地图更新，但不通过回调
    if hasattr(self, 'on_waypoints_changed') and self.on_waypoints_changed:
        # 使用QTimer延迟触发，避免递归
        QTimer.singleShot(50, lambda: self.on_waypoints_changed(self.waypoints))
```

### 技术优势
- **避免UI阻塞**：使用QTimer确保UI响应性
- **防止递归**：分离更新和回调触发
- **异常安全**：完整的异常处理机制
- **信号保护**：自动重连信号连接
- **延迟更新**：避免同步调用导致的死锁
