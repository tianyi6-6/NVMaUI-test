# 1. 文档目标

为 NVMagUI 新增一个与原有功能零耦合、零破坏的「自定义实验-节点工作流」模块，提供对标 ComfyUI 的科研仪器可视化流程搭建与执行能力。

# 2. 背景与约束

## 2.1 项目基线

- 现有项目：`NVMagUI`（金刚石磁探测器自动化实验控制程序）
- 技术栈：Python + PySide6 + pyqtgraph（项目内已有）
- 现有 UI：顶部 Tab、参数配置区、状态区、绘图区、日志区、既有标签页

## 2.2 硬性约束（必须满足）

- 不修改、不破坏原有 8 个标签页的代码、逻辑、行为与界面。
- 新能力全部增量开发，独立目录管理，最小侵入主程序。
- 原有运行流程、设备通信、参数下发、历史功能保持 100% 可用。

# 3. 产品目标

面向科研人员提供低代码实验编排能力，实现：

- 节点拖拽建图
- 连线拓扑与参数配置
- 拓扑顺序执行/单步调试/中断停止
- 复用原设备接口进行控制与采集
- 实时流式绘图
- 工作流保存/加载/导出

# 4. 用户与场景

## 4.1 目标用户

- 实验人员、调试工程师、算法工程师（无需深度编码也可搭建实验流程）

## 4.2 关键场景

- 快速搭建“连接设备 -> 参数配置 -> 采集 -> 计算 -> 绘图 -> 存档”链路
- 复用历史工作流工程进行复现实验
- 通过单步执行定位异常节点与设备响应问题

# 5. 功能需求（Must）

## 5.1 界面嵌入

1. 在顶部 Tab 最右新增标签页：`自定义实验-节点工作流`
2. 新标签页内部为独立工作区，与原有页完全隔离
3. 视觉风格继承项目主题体系（颜色、字体、控件样式）

## 5.2 节点编辑器能力

1. **节点库侧栏**：按分类显示节点（设备连接、参数、PID、CW、频谱、计算、绘图、逻辑、延时等）
2. **画布操作**：拖拽、框选、多选、复制粘贴删除、缩放平移、右键菜单
3. **端口连线**：输入输出端口、类型校验、非法连线拦截、链路可视化
4. **执行控制**：拓扑排序执行、分步调试、单步运行、停止运行、节点状态高亮
5. **设备复用**：节点直接复用原设备接口/参数体系，不重复造通信层
6. **流式绘图**：多通道实时刷新、缓存、坐标自适应，兼容既有数据格式
7. **工程管理**：
   - 新建空白工作流
   - 保存 `.nvm_workflow`
   - 加载历史工程
   - 导出 JSON
   - 导出 Python 脚本
   - 导出实验报告 PDF

## 5.3 非功能需求

- 兼容既有 Python 运行环境与依赖体系
- 画布 200+ 节点可编辑与浏览流畅
- 实时绘图刷新率不低于 10Hz
- 设备指令响应目标 < 100ms（在设备条件满足时）
- 提供节点扩展注册机制，支持后续快速增量

# 6. 界面原型规范

## 6.1 页面结构

- 顶部：原 Tab + 新 Tab（最右）
- 新 Tab 内部分栏：
  - 左：节点库 + 工具栏（新建/保存/加载/运行/停止/清空/导出）
  - 中：无限画布节点编辑区
  - 右：节点属性面板
  - 底：工作流运行日志（与现有日志体系对齐）

## 6.2 状态反馈

- 节点状态：等待/运行中/成功/失败（颜色与图标区分）
- 执行总状态：空闲/执行中/已停止/异常

# 7. 数据与文件规范

## 7.1 `.nvm_workflow` 工程结构（建议）

- 基础元信息：版本、创建时间、修改时间
- 节点列表：`id/type/title/pos/params`
- 连线列表：`from_node/from_port/to_node/to_port`
- 运行配置：执行策略、并发开关、日志级别

## 7.2 导出格式

- JSON：便于审查与版本管理
- Python：便于离线批量执行
- PDF：便于实验归档

# 8. 验收标准

1. 原有 8 个标签页功能回归全部通过
2. 新 Tab 可正常编辑节点与连线
3. 可运行拓扑流程并驱动原设备接口执行
4. 可实时显示流式绘图数据
5. 可保存/加载/导出工作流工程
6. 新增代码目录独立，主程序仅保留必要入口注册
