# 1. 设计目标

在不影响既有 8 个标签页的前提下，新增可插拔的节点工作流系统，实现“可视化建图 + DAG 执行 + 设备控制 + 实时绘图 + 工程管理”。

# 2. 设计原则

- **零侵入**：原页面与核心流程不改动，仅新增 Tab 注册入口。
- **强隔离**：新模块目录独立，内部自管理生命周期。
- **可复用**：复用现有设备接口、参数结构、日志与主题体系。
- **可扩展**：节点注册制，新增节点无需改引擎核心。

# 4. 主程序集成点（唯一允许变更点）

在主 UI 初始化阶段，仅新增：

1. `import workflow_extension.ui.workflow_tab`
2. 创建 `WorkflowTab` 实例
3. `QTabWidget.addTab(workflow_tab, "自定义实验-节点工作流")`

禁止改动原有 8 个标签页构造与逻辑。

# 5. 模块设计

## 5.1 UI 层

- `workflow_tab.py`：主容器，负责左右中布局装配
- `node_palette.py`：节点分类树与拖拽源
- `workflow_canvas.py`：基于 qtpynodeeditor 的画布管理
- `node_property_panel.py`：选中节点参数编辑
- `workflow_toolbar.py`：工程操作与运行控制
- `workflow_log_panel.py`：工作流日志视图，桥接主日志系统

## 5.2 核心层

- `node_registry.py`：节点注册与查询（扩展入口）
- `data_contracts.py`：端口类型定义、连线校验规则
- `dag_executor.py`：拓扑排序、运行调度、中断控制、状态广播
- `execution_context.py`：运行时上下文（设备句柄、缓存、全局参数）
- `workflow_serializer.py`：保存/加载 `.nvm_workflow`
- `workflow_exporter.py`：JSON/Python/PDF 导出

## 5.3 适配层

- `device_adapter.py`：统一桥接既有设备接口（避免节点直接依赖旧代码细节）
- `plot_adapter.py`：统一 pyqtgraph 数据流写入接口
- `theme_adapter.py`：读取并应用既有主题风格
- `logger_adapter.py`：统一日志通道，输出到现有日志区域

# 6. 关键流程设计

## 6.1 节点编辑流程

1. 从节点库拖拽节点到画布
2. 创建默认参数与端口
3. 连线时触发端口类型校验
4. 非法连线提示并阻止提交

## 6.2 执行流程（DAG）

1. 从终端节点回溯构图，检查环路
2. 执行拓扑排序
3. 按拓扑顺序运行节点 `execute(context)`
4. 节点状态事件回推 UI（等待/运行/成功/失败）
5. 任一节点失败时按策略：终止或继续（可配置）
6. 用户点停止时触发中断信号，安全退出

## 6.3 流式绘图流程

1. 采集/计算节点输出结构化数据
2. 进入 `plot_adapter` 统一转换
3. 以定频刷新策略写入 pyqtgraph
4. 缓存窗口按配置裁剪，避免内存膨胀

# 7. 数据模型（建议）

## 7.1 NodeSchema

- `id: str`
- `type: str`
- `title: str`
- `position: tuple[float, float]`
- `params: dict`
- `inputs: list[PortSchema]`
- `outputs: list[PortSchema]`

## 7.2 EdgeSchema

- `from_node: str`
- `from_port: str`
- `to_node: str`
- `to_port: str`

## 7.3 WorkflowSchema

- `meta: dict`
- `nodes: list[NodeSchema]`
- `edges: list[EdgeSchema]`
- `runtime: dict`

# 8. 节点扩展规范

## 8.1 BaseNode 接口约定

- `define_ports() -> PortSpec`
- `default_params() -> dict`
- `validate(params) -> list[str]`
- `execute(context, inputs) -> outputs`

## 8.2 注册规范

新节点仅需：

1. 实现 `BaseNode` 子类
2. 在 `node_registry.py` 注册元信息（名称/分类/图标/类引用）

# 9. 异常与容错

- 节点执行异常：捕获后写日志、节点置红、上抛运行结果
- 设备离线：节点返回可识别错误码，不崩溃 UI
- 配置缺失：使用默认值并提示
- 文件损坏：加载失败时保留当前画布并给出恢复建议

# 10. 性能设计

- 画布渲染：按需更新，避免全量重绘
- 执行线程：执行引擎与 UI 主线程解耦（信号/槽同步状态）
- 绘图刷新：限制刷新频率（10\~20Hz），数据缓存滑窗

# 11. 测试与验收建议

## 11.1 单元测试

- DAG 拓扑排序正确性
- 连线类型校验
- 工作流序列化/反序列化一致性

## 11.2 集成测试

- 设备连接节点到采集节点端到端
- 停止执行与异常中断恢复
- 多节点（200 节点）操作流畅性

## 11.3 回归测试

- 原 8 标签页关键流程冒烟测试（启动、参数下发、采集、日志）
