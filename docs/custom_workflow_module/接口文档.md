## 核心数据模型

### models.py

该模块定义了工作流系统的核心数据结构，所有模型类均使用`@dataclass`装饰器，支持自动序列化。

#### WorkflowNodeModel - 工作流节点模型

| 属性 | 类型 | 说明 |
|------|------|------|
| `node_id` | `str` | 节点唯一标识符，在图中必须唯一 |
| `node_type` | `str` | 节点类型，对应注册表中的节点类型标识 |
| `title` | `str` | 节点显示标题，在UI中显示给用户 |
| `position` | `Tuple[float, float]` | 节点在画布中的坐标位置 (x, y) |
| `params` | `Dict[str, object]` | 节点参数配置，键值对形式存储 |

```python
# 示例用法
node = WorkflowNodeModel(
    node_id="data_source_1",
    node_type="data.source",
    title="数据源",
    position=(100.0, 200.0),
    params={"frequency": 1000, "amplitude": 1.0}
)
```

#### WorkflowEdgeModel - 工作流边模型

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `from_node` | `str` | - | 源节点ID，数据来源节点 |
| `to_node` | `str` | - | 目标节点ID，数据接收节点 |
| `from_port` | `str` | `"out"` | 源端口名称 |
| `to_port` | `str` | `"in"` | 目标端口名称 |

```python
# 示例用法
edge = WorkflowEdgeModel(
    from_node="data_source_1",
    to_node="plot_1",
    from_port="out",
    to_port="in"
)
```

#### WorkflowGraphModel - 工作流图模型

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `version` | `str` | `"1.0"` | 工作流版本号，用于兼容性检查 |
| `name` | `str` | `"Untitled"` | 工作流名称，用户可编辑 |
| `nodes` | `List[WorkflowNodeModel]` | `[]` | 工作流中所有节点的列表 |
| `edges` | `List[WorkflowEdgeModel]` | `[]` | 工作流中所有连接边的列表 |

```python
# 示例用法
workflow = WorkflowGraphModel(
    name="测试工作流",
    nodes=[node1, node2],
    edges=[edge1, edge2]
)
```

## 节点注册系统

### node_registry.py

该模块提供了完整的节点注册和管理机制，支持动态节点注册和类型安全的节点规范定义。

#### NodePortSpec - 节点端口规范

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `name` | `str` | - | 端口名称，在节点中必须唯一 |
| `data_type` | `str` | `"any"` | 端口数据类型，支持: any, float, int, array, bool, trigger |

```python
# 示例用法
input_port = NodePortSpec(name="data_in", data_type="float")
output_port = NodePortSpec(name="result", data_type="array")
```

#### NodeParamSpec - 节点参数规范

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `key` | `str` | - | 参数键名，用于在节点参数字典中标识该参数 |
| `label` | `str` | - | 参数显示标签，在UI中显示给用户 |
| `editor` | `str` | `"text"` | 编辑器类型，支持: text, int, float, select, bool |
| `options` | `List[str]` | `[]` | 选择项列表，当editor为"select"时使用 |
| `minimum` | `float` | `0.0` | 数值最小值，当editor为"int"或"float"时使用 |
| `maximum` | `float` | `999999.0` | 数值最大值，当editor为"int"或"float"时使用 |
| `step` | `float` | `1.0` | 数值步长，当editor为"int"或"float"时使用 |

```python
# 示例用法
param_spec = NodeParamSpec(
    key="frequency",
    label="频率",
    editor="float",
    minimum=0.1,
    maximum=10000.0,
    step=0.1
)
```

#### NodeSpec - 节点完整规范

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `node_type` | `str` | - | 节点类型标识符，必须唯一 |
| `title` | `str` | - | 节点显示标题，在UI中显示给用户 |
| `category` | `str` | - | 节点分类，用于在节点库中分组显示 |
| `default_params` | `Dict[str, object]` | `{}` | 默认参数配置 |
| `input_ports` | `List[NodePortSpec]` | `[NodePortSpec("in", "any")]` | 输入端口列表 |
| `output_ports` | `List[NodePortSpec]` | `[NodePortSpec("out", "any")]` | 输出端口列表 |
| `param_specs` | `List[NodeParamSpec]` | `[]` | 参数规范列表 |
| `executor` | `Callable` | `None` | 节点执行函数，定义节点的业务逻辑 |

```python
# 示例用法
def my_executor(context, node, inputs):
    return {"result": sum(inputs)}

spec = NodeSpec(
    node_type="math.sum",
    title="求和",
    category="数学运算",
    default_params={"scale": 1.0},
    param_specs=[
        NodeParamSpec("scale", "缩放", "float", minimum=0.1, maximum=10.0)
    ],
    executor=my_executor
)
```

#### NodeRegistry - 节点注册器

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `__init__()` | - | `None` | 初始化节点注册器，创建空的节点规范字典 |
| `register(spec)` | `spec: NodeSpec` | `None` | 注册节点规范，如果节点类型已存在则覆盖 |
| `get(node_type)` | `node_type: str` | `NodeSpec` | 获取节点规范，当节点类型不存在时抛出KeyError |
| `all_specs()` | - | `List[NodeSpec]` | 获取所有节点规范列表 |
| `grouped()` | - | `Dict[str, List[NodeSpec]]` | 按分类获取节点规范，便于UI分类显示 |

```python
# 示例用法
registry = NodeRegistry()
registry.register(spec)
all_nodes = registry.all_specs()
math_nodes = registry.grouped()["数学运算"]
```

## 工作流执行引擎

### engine.py

该模块实现了工作流的核心执行逻辑，包括拓扑排序、异步执行和状态管理。

#### WorkflowExecutor - 工作流执行器

##### 信号定义

| 信号 | 参数 | 说明 |
|------|------|------|
| `node_started` | `str` | 节点开始执行信号，参数为节点ID |
| `node_finished` | `str, object` | 节点执行完成信号，参数为节点ID和执行结果 |
| `node_failed` | `str, str` | 节点执行失败信号，参数为节点ID和错误信息 |
| `run_finished` | - | 工作流执行完成信号 |

##### 方法

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `__init__(registry, parent=None)` | `registry: NodeRegistry`, `parent: QObject` | `None` | 初始化工作流执行器 |
| `stop()` | - | `None` | 请求停止工作流执行 |
| `run(graph, context)` | `graph: WorkflowGraphModel`, `context: Dict[str, Any]` | `None` | 执行工作流，按拓扑顺序执行所有节点 |
| `_topological_order(graph)` | `graph: WorkflowGraphModel` | `List[WorkflowNodeModel]` | 静态方法，计算工作流的拓扑排序 |

```python
# 示例用法
executor = WorkflowExecutor(registry)

# 连接信号
executor.node_started.connect(on_node_started)
executor.node_finished.connect(on_node_finished)
executor.run_finished.connect(on_run_finished)

# 执行工作流
context = {"app": app_instance, "plot_callback": plot_function}
executor.run(workflow, context)
```

## 序列化工具

### serializer.py

该模块提供了工作流的序列化和反序列化功能，支持加密存储和多种导出格式。

#### 全局常量

| 常量 | 类型 | 值 | 说明 |
|------|------|-----|------|
| `_KEY` | `bytes` | `b"NVMAG_WORKFLOW_KEY"` | XOR加密密钥 |

#### 函数

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `_xor_crypt(data)` | `data: bytes` | `bytes` | XOR加密/解密算法，加密和解密使用相同操作 |
| `save_workflow(graph, file_path)` | `graph: WorkflowGraphModel`, `file_path: str` | `None` | 保存工作流到加密文件，使用XOR+Base64 |
| `load_workflow(file_path)` | `file_path: str` | `WorkflowGraphModel` | 从加密文件加载工作流 |
| `export_json(graph, file_path)` | `graph: WorkflowGraphModel`, `file_path: str` | `None` | 导出工作流为纯JSON格式 |
| `export_python(graph, file_path)` | `graph: WorkflowGraphModel`, `file_path: str` | `None` | 导出工作流为Python脚本格式 |

```python
# 示例用法
# 保存工作流
save_workflow(workflow, "my_workflow.nvw")

# 加载工作流
workflow = load_workflow("my_workflow.nvw")

# 导出JSON
export_json(workflow, "workflow.json")

# 导出Python脚本
export_python(workflow, "workflow.py")
```

## 撤销系统

### undo_system.py

该模块实现了完整的撤销重做功能，基于命令模式设计，支持工作流编辑操作的历史管理。

#### 命令基类

##### WorkflowCommand - 工作流命令基类

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `__init__(scene, description)` | `scene: WorkflowScene`, `description: str` | `None` | 初始化命令 |
| `execute()` | - | `None` | 执行命令（子类实现） |
| `undo()` | - | `None` | 撤销命令（子类实现） |
| `redo()` | - | `None` | 重做命令（默认调用execute） |

#### 具体命令类

| 命令类 | 说明 | 主要属性 |
|--------|------|----------|
| `AddNodeCommand` | 添加节点命令 | `node_item`, `pos` |
| `DeleteNodesCommand` | 删除节点命令 | `node_items`, `edges` |
| `AddEdgeCommand` | 添加连接命令 | `edge_item` |
| `RemoveEdgeCommand` | 移除连接命令 | `edge_item` |
| `MoveNodesCommand` | 移动节点命令 | `node_items`, `old_positions`, `new_positions` |

#### WorkflowUndoStack - 撤销栈

##### 属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `undo_stack` | `List[WorkflowCommand]` | 撤销命令栈 |
| `redo_stack` | `List[WorkflowCommand]` | 重做命令栈 |
| `max_stack_size` | `int` | 最大栈大小限制 |

##### 方法

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `__init__(scene, max_size=50)` | `scene: WorkflowScene`, `max_size: int` | `None` | 初始化撤销栈 |
| `push(command)` | `command: WorkflowCommand` | `None` | 推送命令到撤销栈 |
| `undo()` | - | `bool` | 执行撤销操作 |
| `redo()` | - | `bool` | 执行重做操作 |
| `clear()` | - | `None` | 清空撤销栈 |
| `can_undo()` | - | `bool` | 检查是否可以撤销 |
| `can_redo()` | - | `bool` | 检查是否可以重做 |

# 示例用法
undo_stack = WorkflowUndoStack(scene)

# 执行可撤销操作

undo_stack.push(command)

# 撤销操作

    undo_stack.undo()

### 自定义节点开发

```python
from workflow_extension.node_registry import NodeSpec, NodeParamSpec, NodePortSpec

def custom_executor(context, node, inputs):
    """自定义节点执行器"""
    # 获取参数
    multiplier = node.params.get("multiplier", 1.0)
    
    # 处理输入数据
    result = []
    for input_data in inputs:
        if isinstance(input_data, (int, float)):
            result.append(input_data * multiplier)
        elif isinstance(input_data, dict) and "y" in input_data:
            result.append(input_data["y"] * multiplier)
    
    return {"result": result, "multiplier": multiplier}

# 创建节点规范
custom_spec = NodeSpec(
    node_type="custom.multiplier",
    title="数据放大",
    category="自定义",
    default_params={"multiplier": 2.0},
    param_specs=[
        NodeParamSpec(
            key="multiplier",
            label="放大倍数",
            editor="float",
            minimum=0.1,
            maximum=10.0,
            step=0.1
        )
    ],
    input_ports=[NodePortSpec("in", "any")],
    output_ports=[NodePortSpec("out", "array")],
    executor=custom_executor
)

# 注册自定义节点
registry.register(custom_spec)
```

### 工作流执行

```python
from workflow_extension.engine import WorkflowExecutor

# 创建执行器
executor = WorkflowExecutor(registry)

# 连接信号
def on_node_started(node_id):
    print(f"节点 {node_id} 开始执行")

def on_node_finished(node_id, result):
    print(f"节点 {node_id} 执行完成: {result}")

def on_run_finished():
    print("工作流执行完成")

executor.node_started.connect(on_node_started)
executor.node_finished.connect(on_node_finished)
executor.run_finished.connect(on_run_finished)

# 准备执行上下文
context = {
    "app": app_instance,
    "plot_callback": lambda data: print(f"绘图数据: {data}")
}

# 执行工作流
executor.run(workflow, context)
```

## 注意事项

### 开发规范

1. **节点执行器签名**: 所有节点执行器必须遵循 `def executor(context, node, inputs) -> Dict[str, Any]` 签名
2. **线程安全**: 节点执行器应在主线程中运行，避免多线程操作
3. **错误处理**: 节点执行器应妥善处理异常，避免导致整个工作流崩溃
4. **参数验证**: 节点执行器应验证输入参数的有效性

### 性能优化

1. **延时控制**: 避免在节点执行器中使用过长的延时操作
2. **内存管理**: 及时释放大型数据对象，避免内存泄漏
3. **UI响应**: 在长时间操作中适当调用 `QCoreApplication.processEvents()` 保持UI响应
