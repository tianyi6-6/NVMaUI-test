# NVMagUI 自定义实验节点工作流扩展模块

## 项目介绍

NVMagUI 自定义实验节点工作流扩展模块是一个对标 ComfyUI 的拖拽式实验工作流系统，旨在让科研人员以低代码方式完成实验流程编排、执行与复用。该模块在不改动现有核心业务的前提下，为 NVMagUI 增加了可视化工作流能力，大幅提升实验流程搭建与调整的效率。

### 为什么使用此模块

- **效率提升**：实验流程搭建与调整从"改代码/多页面切换"变成"拖拽编排"
- **可复用**：工作流工程可沉淀、共享、复现实验
- **可观测**：节点状态和运行日志可视化，排障更快
- **可扩展**：新节点注册即用，后续迭代成本低

## 功能特性

- **可视化建图**：拖拽节点、连线，直观构建实验流程
- **DAG 拓扑执行**：支持完整运行、单步调试、停止中断
- **设备控制**：复用既有设备接口，打通设备控制与数据采集
- **实时绘图**：支持实时流式数据可视化
- **工程管理**：支持工作流保存、加载、导出（JSON/Python/PDF）
- **节点扩展**：注册制节点系统，支持自定义节点开发
- **主题适配**：自动适配现有主题风格
- **日志集成**：统一日志通道，输出到现有日志区域

## 核心模块

1. **UI 层**：负责用户交互界面，包括画布、节点面板、属性面板等
2. **核心层**：实现工作流执行引擎、节点管理、序列化等核心功能
3. **适配层**：桥接现有系统，复用设备接口、日志、主题和绘图能力
4. **节点层**：提供各类功能节点，包括设备控制、逻辑处理、数据处理等

### 关键流程

#### 节点编辑流程

1. 从节点库拖拽节点到画布
2. 创建默认参数与端口
3. 连线时触发端口类型校验
4. 非法连线提示并阻止提交

#### 执行流程（DAG）

1. 从终端节点回溯构图，检查环路
2. 执行拓扑排序
3. 按拓扑顺序运行节点 `execute(context)`
4. 节点状态事件回推 UI（等待/运行/成功/失败）
5. 任一节点失败时按策略：终止或继续（可配置）
6. 用户点停止时触发中断信号，安全退出

#### 流式绘图流程

1. 采集/计算节点输出结构化数据
2. 进入 `plot_adapter` 统一转换
3. 以定频刷新策略写入 pyqtgraph
4. 缓存窗口按配置裁剪，避免内存膨胀

## 安装与集成

### 集成到主程序

在主 UI 初始化阶段，仅需添加以下代码：

```python
# 导入工作流标签页
import workflow_extension.ui.workflow_tab

# 创建工作流标签页实例
workflow_tab = workflow_extension.ui.workflow_tab.WorkflowTab()

# 添加到主标签页控件
QTabWidget.addTab(workflow_tab, "自定义实验-节点工作流")
```

**注意**：禁止改动原有 8 个标签页构造与逻辑。

## 快速开始

1. **创建新工作流**：点击工具栏的"新建"按钮

## 核心概念

### 节点 (Node)

节点是工作流的基本执行单元，包含输入端口、输出端口和参数配置。每个节点实现特定的功能，如设备控制、数据处理等。

### 端口 (Port)

端口是节点间数据传输的接口，分为输入端口和输出端口。端口有类型约束，确保数据传输的正确性。

### 边 (Edge)

边表示节点间的连接关系，定义了数据从一个节点的输出端口流向另一个节点的输入端口。

### 工作流 (Workflow)

工作流由节点和边组成的有向无环图 (DAG)，表示完整的实验流程。

### 执行上下文 (Execution Context)

执行上下文包含运行时所需的资源，如设备句柄、缓存、全局参数等。

## 使用指南

### 节点操作

- **拖拽节点**：从左侧节点面板拖拽到画布
- **移动节点**：点击节点并拖动到新位置
- **删除节点**：选中节点后按 Delete 键，或右键菜单选择删除
- **复制节点**：选中节点后按 Ctrl+C，然后按 Ctrl+V 粘贴
- **选择多个节点**：按住 Shift 键并点击多个节点
- **对齐节点**：选中多个节点后，使用右键菜单的对齐选项

### 连线操作

- **创建连线**：点击源节点输出端口，拖动到目标节点输入端口
- **删除连线**：选中连线后按 Delete 键，或右键菜单选择删除
- **弯曲连线**：点击并拖动连线中间部分调整路径

### 运行控制

- **运行**：执行整个工作流
- **单步**：执行当前选中节点
- **停止**：中断正在执行的工作流
- **暂停**：暂停工作流执行（可恢复）

### 视图操作

- **缩放**：使用鼠标滚轮或工具栏缩放按钮
- **平移**：按住鼠标中键拖动画布
- **重置视图**：点击工具栏的"重置视图"按钮
- **居中选中**：选中节点后，点击右键菜单的"居中选中"

## 节点类型

### 设备节点

- **设备连接**：建立与设备的连接
- **参数设置**：设置设备参数
- **PID控制**：实现 PID 控制逻辑
- **连续波**：控制连续波输出
- **光谱**：采集和处理光谱数据

### 逻辑节点

- **条件**：根据条件执行不同分支
- **延迟**：添加时间延迟
- **循环**：实现循环执行逻辑

### 数据节点

- **数据转换**：转换数据格式或进行数据处理
- **流式绘图**：实时绘制数据曲线

## 工作流管理

### 保存与加载

- **保存**：将工作流保存为 `.nvm_workflow` 文件
- **加载**：从 `.nvm_workflow` 文件加载工作流
- **另存为**：将当前工作流保存为新文件

### 导出功能

- **导出为 JSON**：将工作流导出为 JSON 格式
- **导出为 Python**：生成可执行的 Python 代码
- **导出为 PDF**：生成工作流图表的 PDF 文档

## 扩展开发

### 创建自定义节点

1. **继承 BaseNode**：创建一个继承自 `BaseNode` 的新类
2. **实现必要方法**：
   - `define_ports()`：定义节点的输入输出端口
   - `default_params()`：提供默认参数
   - `validate(params)`：验证参数有效性
   - `execute(context, inputs)`：实现节点执行逻辑
3. **注册节点**：在 `node_registry.py` 中注册节点元信息

### 节点注册示例

```python
# 在 node_registry.py 中添加
node_registry.register_node(
    type="my_custom_node",
    title="我的自定义节点",
    category="自定义",
    icon="path/to/icon.png",
    node_class=MyCustomNode
)
```

### 适配器开发

如需扩展适配器以支持新的设备或功能，可在 `adapters` 目录下创建新的适配器类。

## 故障排除

### 常见问题

1. **节点执行失败**
   - 检查节点参数是否正确
   - 检查设备连接状态
   - 查看日志面板获取详细错误信息
2. **工作流加载失败**
   - 检查文件是否损坏
   - 确保使用的是兼容版本的工作流文件
3. **性能问题**
   - 减少节点数量或优化节点逻辑
   - 调整绘图刷新频率
   - 关闭不必要的实时绘图
4. **设备连接问题**
   - 检查设备驱动是否正确安装
   - 验证设备地址和端口设置
   - 尝试重新连接设备

### 日志查看

工作流执行过程中的日志会显示在日志面板中，同时也会写入 NVMagUI 的主日志系统。遇到问题时，请查看日志获取详细信息。

## 性能优化

### 画布性能

- **按需更新**：画布仅在必要时重绘
- **节点分组**：对于复杂工作流，使用节点分组减少视觉复杂度
- **缩放级别**：在合适的缩放级别下操作，避免过度缩放

### 执行性能

- **执行线程**：执行引擎与 UI 主线程解耦，避免阻塞界面
- **批量处理**：对于数据密集型操作，使用批量处理减少开销
- **缓存策略**：合理使用缓存，避免重复计算

### 绘图性能

- **刷新频率**：限制绘图刷新频率（10\~20Hz）
- **数据缓存**：使用滑动窗口缓存，避免内存膨胀
- **按需绘图**：仅绘制可见区域的数据
