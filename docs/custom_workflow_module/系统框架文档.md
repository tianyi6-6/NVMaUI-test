# 自定义实验-节点工作流系统框架文档

## 1. 系统概述

### 1.1 系统简介
自定义实验-节点工作流系统是一个基于PyQt6的可视化编程框架，专为科学实验流程设计。系统采用节点化编程模式，通过拖拽节点、配置参数、连接端口来构建复杂的实验工作流。

### 1.2 设计目标
- **可视化编程**：提供直观的图形界面，降低编程门槛
- **模块化设计**：支持插件式节点扩展，便于功能扩展
- **实时执行**：支持工作流实时执行和状态监控
- **数据可视化**：内置实时绘图功能，支持实验数据可视化
- **可扩展性**：提供标准接口规范，支持自定义节点开发

### 1.3 技术栈
- **UI框架**：PySide6 (Qt6)
- **绘图库**：pyqtgraph
- **数据结构**：dataclass, typing
- **序列化**：JSON + XOR加密
- **日志系统**：logging

## 3. 接口规范

### 3.1 节点接口规范

#### 3.1.1 节点规范接口 (NodeSpec)
```python
@dataclass
class NodeSpec:
    node_type: str          # 节点类型标识符
    title: str            # 节点显示标题
    category: str         # 节点分类
    default_params: Dict[str, object] = field(default_factory=dict)
    input_ports: List[NodePortSpec] = field(default_factory=lambda: [NodePortSpec("in", "any")])
    output_ports: List[NodePortSpec] = field(default_factory=lambda: [NodePortSpec("out", "any")])
    param_specs: List[NodeParamSpec] = field(default_factory=list)
    executor: Callable = None
```

#### 3.1.2 端口规范接口 (NodePortSpec)
```python
@dataclass
class NodePortSpec:
    name: str            # 端口名称
    data_type: str = "any"  # 数据类型: any, float, int, array, bool, trigger
```

#### 3.1.3 参数规范接口 (NodeParamSpec)
```python
@dataclass
class NodeParamSpec:
    key: str             # 参数键名

    options: List[str] = field(default_factory=list)  # 选择项列表
    minimum: float = 0.0     # 最小值
    maximum: float = 999999.0  # 最大值
    step: float = 1.0        # 步长
```

#### 4.2.1 端口数据传递
- **输入端口**：接收上游节点的输出数据

```python
def custom_node_executor(context, node, inputs):
    """自定义节点执行器"""
    try:
        # 获取参数
        param1 = node.params.get("param1", "default_value")
        
        # 处理输入数据
        input_data = inputs[0] if inputs else None
        
        # 执行业务逻辑
        result_data = process_data(input_data, param1)
        
        # 返回结果
        return {
            "output_key": result_data,
            "status": "success"
        }
    except Exception as e:
        logging.error(f"节点执行失败: {e}")
        return {
            "error": str(e),
            "status": "error"
        }
```
