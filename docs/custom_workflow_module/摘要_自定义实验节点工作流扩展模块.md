# 摘要：NVMagUI 自定义实验节点工作流扩展模块

## 1. 项目一句话

在不改动现有核心业务的前提下，为 NVMagUI 增加一个对标 ComfyUI 的拖拽式实验工作流模块，让科研人员以低代码方式完成实验流程编排、执行与复用。

## 2. 为什么现在做

- 当前流程主要依赖固定页面操作，流程复用与快速试验效率有限。
- 不同实验场景切换成本高，参数组合与链路重构效率低。
- 需要统一的“可视化编排 + 执行 + 数据绘图 + 归档”能力，支撑后续持续迭代。

## 3. 目标与边界

## 3.1 建设目标

- 提供拖拽节点、连线、拓扑执行、单步调试、停止中断
- 复用既有设备接口，打通设备控制与数据采集
- 支持实时流式绘图与工程保存/加载/导出

## 3.2 约束边界

- 原有 8 个标签页完全不动
- 新模块独立目录、独立生命周期
- 主程序仅新增 Tab 注册点

## 4. 方案概览

- UI：新增 `自定义实验-节点工作流` 标签页
- 引擎：DAG 拓扑执行器（支持运行、单步、停止、错误高亮）
- 适配：设备/日志/主题/绘图通过 adapter 统一桥接旧系统
- 文件：`.nvm_workflow` + JSON/Python/PDF 导出

## 5. 预期收益

- **效率提升**：实验流程搭建与调整从“改代码/多页面切换”变成“拖拽编排”
- **可复用**：工作流工程可沉淀、共享、复现实验
- **可观测**：节点状态和运行日志可视化，排障更快
- **可扩展**：新节点注册即用，后续迭代成本低

## 6. 里程碑（12周）

- **M1（1-4周）**：架构落地 + 交互原型
- **M2（5-8周）**：执行引擎 + 设备联动 + 保存加载
- **M3（9-12周）**：导出能力 + 性能优化 + 回归发布

## 7. 成功标准（KPI）

- 原有 8 标签页回归通过率 100%
- 新模块核心链路可用率 100%（建图/运行/绘图/保存/加载/导出）
- 200 节点编辑场景可用，绘图刷新率不低于 10Hz
- 严重缺陷（P0/P1）在发布前清零

## 8. 主要风险与对策

- **设备联调波动**：提前建立联调窗口和模拟回放数据
- **性能风险**：执行引擎与 UI 解耦，绘图刷新节流
- **边界污染风险**：严格限定主程序改动点，增量模块自闭环

## 9. 资源建议

- 周期：12 周
- 测试资源：真实设备 + 回放数据 + 自动化冒烟脚本

## 10. 立项结论

该项目是对现有平台能力的“低风险高价值”增强：技术可行、边界可控、收益明确，建议按 12 周计划立项实施。
